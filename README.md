# NightShift City 3D

A Unity 3D prototype of an autonomous city-cleaning robot simulation built with primitive shapes, procedural scene generation, and a live HUD.

---

## Description

NightShift City 3D is a top-down 3D simulation where CleanerBots autonomously navigate a procedurally generated city, locate trash, claim it exclusively, and clean it up — all while a live HUD tracks city health in real time.

Built as a learning project to practise Unity fundamentals: component architecture, editor tooling, runtime UI generation, and clean beginner-friendly C#.

---

## Features

- **Autonomous CleanerBots** — find the nearest unclaimed trash, move toward it, and destroy it on contact
- **Exclusive trash claiming** — no two robots chase the same piece of trash
- **Procedural city builder** — one-click Editor tool generates ground, roads, buildings, robots, spawner, and lighting
- **Live city HUD** — displays City Health, Active Trash, Total Spawned/Cleaned, and Robot count; colour-coded health bar (green → yellow → red)
- **Trash spawner** — continuously spawns trash at random positions on a configurable timer
- **Primitive-only visuals** — entire scene built from Unity cubes, spheres, and cylinders (no external assets)
- **Robot name labels** — billboard text labels above each CleanerBot
- **Fully code-driven UI** — Canvas, panel, and TMP text created at runtime; no manual Unity Editor wiring required

## How to Run

1. Clone or download this repository
2. Open the project in **Unity 6** (Universal Render Pipeline)
3. Open `Assets/Scenes/SampleScene`
4. In the Unity top menu click: **Tools → NightShift City → Build Basic Scene**
5. Press **Play**

The scene will populate automatically — no manual Inspector setup required.

## Project Structure

```
Assets/
├── Editor/
│   └── NightShiftCityBuilder.cs   # One-click scene generator
├── Prefabs/
│   └── Trash.prefab               # Auto-generated by the builder
├── Scenes/
│   └── SampleScene.unity
├── CityManager.cs                 # Singleton — tracks all city stats
├── CityUI.cs                      # Self-building runtime HUD
├── RobotController.cs             # Autonomous robot movement and cleaning
├── RobotLabel.cs                  # Billboard label above each robot
├── TrashItem.cs                   # Claim/release logic per trash object
└── TrashSpawner.cs                # Timed trash spawning
```

---

## How the Simulation Works

```
TrashSpawner  →  spawns Trash prefab (tagged "Trash", has TrashItem)
RobotController  →  FindGameObjectsWithTag("Trash")
                 →  finds nearest unclaimed → calls TrashItem.Claim()
                 →  MoveTowards target (X/Z plane only)
                 →  on arrival → Destroy(trash) → CityManager.RegisterTrashCleaned()
CityManager   →  tracks stats → CityUI reads them every frame
```
